    Key Features

    • Signal-Driven Resource: Uses an Angular signal() to drive the resource() reloads whenever the search query changes.
    • Abortable HTTP Requests: Each search request is abortable to prevent race conditions when typing quickly.
    • Dynamic UI Updates: Users list updates automatically in real-time as you type, without manual refresh.
    • Local Resource Updates: Supports adding a new user to the list or clearing the resource without hitting the API again.
    • Loading & Error Handling: Shows a loading indicator while fetching users and a friendly error message if the request fails.
